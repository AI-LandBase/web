## 3. 訴求ゴール / CTA

### 主目的（親 Issue #18）
- A. 営業・受注獲得（観光業事業者からの問い合わせ獲得）
- B. 事業の信頼性・存在証明（指名検索でのプレゼンス確立）

### 主 CTA
**「お気軽にメールでご相談ください」**（mailto: `info@ai-landbase.jp`）に確定。

- 配置: グローバルナビ右上の常駐 CTA、ヒーロー、各セクション末尾、フッター
- 動線: クリックでメーラー起動、件名テンプレート（例: 「サービスに関するお問い合わせ」）をプリセット
- 振り分け方針: `info@` 一本に集約、件名で社内振り分け（サービス相談 / 取材・メディア / パートナー連携 / 採用 / その他）
- スパム対策: メールアドレスを **JavaScript で動的生成** し、HTML に直接埋め込まない
- フォーム不採用の理由: コーポレート訪問者は「ふわっと相談したい」フェーズが多く自由記述の方が会話を始めやすい / 中小経営者にメールは自然 / LP 側に高intent向けフォームが既にある / 実装・運用コスト最小
- **電話番号は掲載しない**: コーポレートサイトでは電話を出さず、連絡手段はメールのみに統一する

### 副 CTA
- 各プラン詳細ページの参照
- 宿泊税対応 LP（lodging-tax.ai-landbase.jp）への遷移（AI Suite Server プラン詳細から）

## 6. 公開先・技術前提

### ドメイン
- **本番**: `ai-landbase.jp`（apex）
- 既存 `lodging-tax.ai-landbase.jp` と同じ Caddy リバースプロキシ配下に追加
- HTTPS は Let's Encrypt 自動化（既存構成踏襲）
- **前提確認（Phase 7 で別 Issue 起票）**: apex `ai-landbase.jp` の DNS A レコード設定状況、Caddy 設定での apex ハンドリング、既存サブドメインへの影響範囲を確認する

### 技術スタック
- Next.js 15（App Router）/ TypeScript / Tailwind CSS v4
- 実装コードは `src/` 配下に直接配置（`projects/{name}/` には入れない）
- DB / 認証 / Prisma は **コーポレートサイトでは不要**（フォーム廃止のため）。リポ自体には Prisma + PostgreSQL + NextAuth が同居しているが、本サイトでは利用しない

### 既存 LP との関係
- 宿泊税対応 LP: https://lodging-tax.ai-landbase.jp/lp/okinawa-lodging-tax/（公開済み）
- コーポレート → LP: AI Suite Server プラン詳細から「宿泊税対応の詳細はこちら」リンク
- LP → コーポレート: 信頼性確認用に LP から `ai-landbase.jp` への戻り導線
- 配信構成: 同 Caddy リバースプロキシ配下に並行配置（apex `ai-landbase.jp` ／ サブドメイン `lodging-tax.ai-landbase.jp`）

### お問い合わせ実装方針
- **方式**: mailto: ベース（フォーム不採用）
- **表示**: メールアドレスを JavaScript で動的生成し、生 `mailto:` を HTML に埋め込まない（簡易スパムボット向け対策。高度なクローラには無効。被害が顕在化したら Cloudflare Email Obfuscation 等を追加検討）
- **件名プリセット**: クリック時に問い合わせ種別を含む件名テンプレートを付与（例: `[AI.LandBase] サービスに関するお問い合わせ`）
- **受信**: `info@ai-landbase.jp` で受信、社内で件名により振り分け
- **DB / 管理画面 / Prisma モデル / Server Action は不要**
- 親 Issue #18 の「Phase 6-5: お問い合わせフォーム実装」は **mailto: 実装＋スパム対策のみ** にスコープ縮小（子 Issue 起票時にタイトルも調整）
