## ペルソナ A: 宮城 健一（みやぎ けんいち）— 宿泊事業者・宿泊税対応が急務

| 項目 | 内容 |
|------|------|
| **属性** | 52歳男性。沖縄県名護市でペンション（客室8室）を経営して18年。妻と従業員2名の小規模経営。PC は予約管理と経理で日常的に使うが、IT システムの導入経験はほぼなし。スマートフォンは LINE・電話・OTA 管理アプリ程度。デジタルリテラシーは中〜低。 |
| **行動トリガー** | ① 令和9年2月施行の宿泊税への対応が迫っている。同業者の集まりで「AI で対応できるらしい」と聞いた。② 沖縄県の補助金（上限200万円・10/10補助）の申請期限が近づき、「実質タダならやりたい」と思い始めた。③ 「沖縄 宿泊税 対応 システム」で検索 → LP（lodging-tax.ai-landbase.jp）に到達 → 会社概要を確認するためにコーポレートサイトへ遷移。 |
| **信頼シグナル** | ① **地元企業であること**: 沖縄県今帰仁村に実際に拠点がある点が最大の安心材料。「本土の IT 企業に任せて大丈夫か」という不安の解消。② **自社宿泊施設（Ikigai Stay）での実証**: 「自分たちで使っている」事実。③ **補助金活用の実績・サポート**: 補助金申請の手伝いまでしてくれるのか。④ **価格の明示**: いくらかかるか事前にわかること。⑤ **代表者や担当者の顔が見える**: 誰が来るのかわかる安心感。 |
| **情報フィルター** | **数字と実例**が刺さる。「導入コスト○○円 → 補助金で実質0円」「チェックイン時の宿泊税徴収が自動化」など、具体的な金額と業務変化が判断材料。技術的な説明（AI アルゴリズム、LLM 等）は響かない。「うちみたいな小さい宿でも使えるの？」に答える事例が最優先。 |
| **主要な懸念** | ① **費用**: 200万円の機器は高額。補助金が本当に使えるのか不安。② **使いこなせるか**: IT に詳しくない自分とスタッフで運用できるのか。③ **サポート体制**: 導入後に困ったとき、すぐ相談できるか。本土の会社だと電話しても対応が遅いイメージ。④ **本当に必要か**: 宿泊税対応だけなら既存 PMS のアップデートで済むのでは？AI Suite の他機能は自分に必要なのか。⑤ **会社の信頼性**: 設立間もない会社に大丈夫か。 |
| **対応セクション** | **ヒーロー**（「沖縄の宿泊業を AI で支える地元パートナー」系のメッセージで第一印象を掴む）→ **サービス紹介**（AI Suite Server プランの価格・内容を明示。補助金活用時の実質負担を併記）→ **会社案内**（所在地・代表者・Ikigai Stay の写真で「地元の会社」を証明）→ **CTA**（メールで気軽に相談。件名テンプレートで障壁を下げる） |

### 想定フロー

```
検索「沖縄 宿泊税 対応」
  → 宿泊税LP（lodging-tax.ai-landbase.jp）
    → 「この会社どんな会社？」→ コーポレートサイトへ
      → ヒーロー → サービス（価格確認）→ 会社案内（信頼確認）
        → メールで問い合わせ
```

## ペルソナ B: 比嘉 真由美（ひが まゆみ）— 飲食事業者・業務効率化が課題

| 項目 | 内容 |
|------|------|
| **属性** | 38歳女性。沖縄県本部町で沖縄料理レストラン（席数40）を経営して5年。夫と従業員4名（うちパート2名）。Instagram での集客に力を入れており、スマートフォンの活用度は高い。PC は経理ソフト（freee）で使う程度。デジタルリテラシーは中程度。 |
| **行動トリガー** | ① 人手不足が深刻化。シフト管理・在庫発注・経理を自分一人でやっており、営業に集中できない。② 観光協会の勉強会で AI.LandBase の紹介を聞いた。「AI で経理や在庫管理を自動化できる」に興味を持った。③ 「AI.LandBase」で指名検索 → コーポレートサイトに直接到達。 |
| **信頼シグナル** | ① **freee 連携の明示**: 今使っている経理ソフトと連携できるなら導入ハードルが低い。② **月額5万円のスタンダードプラン**: 買い切り200万円は無理だが、月額なら検討できる金額感。③ **地元の勉強会での紹介**: 観光協会のお墨付きがある安心感。④ **飲食業での活用事例**: 宿泊業の事例ばかりだと「うちは違う」と感じる。飲食業に使えることが明確にわかる情報が欲しい。 |
| **情報フィルター** | **ストーリーと共感**が刺さる。「同じ悩みを持つ飲食店オーナーが AI で月○時間の業務削減に成功」といった共感型の事例。数字も重要だが、「自分と似た境遇の人が使っている」ことが決め手。図解やイラストでの説明がテキスト大量より好まれる。 |
| **主要な懸念** | ① **月額コスト**: 月5万円は飲食店の利益率を考えると安くない。投資対効果が見えないと踏み切れない。② **飲食業への適用性**: 宿泊業向けの印象が強いが、自分の店にも本当に使えるのか。③ **導入の手間**: 忙しい営業の合間に導入作業をする余裕がない。どのくらい手間がかかるのか。④ **解約のしやすさ**: 合わなかったときにすぐやめられるか。 |
| **対応セクション** | **ヒーロー**（「観光業の"やらなきゃ"を AI が引き受ける」系の業務効率化訴求で共感を得る）→ **サービス紹介**（スタンダードプランの月額・サービス内容。AI Suite の中から飲食業に関連するツール（AccountingAI / InventoryAI / OperationAI）をピックアップ）→ **AI Suite 一覧**（10ツールの概要で「こんなこともできるのか」と可能性を感じてもらう）→ **CTA**（メールで相談。「まずは月額プランの詳細を聞きたい」程度の軽い問い合わせ） |

### 想定フロー

```
観光協会の勉強会 or 知人の紹介
  → 「AI.LandBase」で指名検索
    → コーポレートサイト（トップ）
      → ヒーロー → サービス（月額プラン確認）→ AI Suite 一覧
        → メールで問い合わせ
```

## ペルソナ C: 島袋 浩二（しまぶくろ こうじ）— 自治体・観光協会担当者

| 項目 | 内容 |
|------|------|
| **属性** | 45歳男性。沖縄県北部の市町村役場・観光振興課の主任。地域の観光業振興と DX 推進を担当。IT 補助金・観光振興施策の情報収集が日常業務。PC リテラシーは高い。プレゼン資料や報告書作成に慣れている。 |
| **行動トリガー** | ① 沖縄県の宿泊税施行に伴い、管内の宿泊事業者への DX 支援策を検討中。「地元に AI 関連の企業はないか」と調査。② 県の補助金制度の周知先として、導入支援ができる地元企業をリストアップする必要がある。③ 「沖縄 AI 観光 企業」「AI.LandBase」で検索 → コーポレートサイトに到達。 |
| **信頼シグナル** | ① **会社の正式情報**: 法人番号・所在地・代表者・設立日・資本金が明記されていること。公的機関として取引先の実在確認が必須。② **事業内容の明確さ**: 何を提供している会社なのかが端的にわかること。上長や議会への説明材料になる。③ **地域貢献・サステナビリティの姿勢**: 行政との協働に前向きであること。④ **実績**: 具体的な導入事例があれば報告書に記載できる。 |
| **情報フィルター** | **体系的な情報と公式データ**が刺さる。ミッション・ビジョン・事業概要・会社概要が整理されていること。パンフレット的にまとまった情報を求める。感情的な訴求よりも客観的な事実。PDF でダウンロードできる会社案内があると理想的。 |
| **主要な懸念** | ① **会社の継続性**: 設立間もないスタートアップに公的事業を委託できるか。資本金300万円は行政の取引先としては小規模。② **実績の少なさ**: 導入実績が少ない段階で自治体として推薦できるか。③ **対応範囲**: 管内の全事業者（数十〜数百社）をカバーできるキャパシティがあるか。④ **中立性**: 特定企業を推薦することの公平性。 |
| **対応セクション** | **会社案内**（法人情報・代表者・所在地・事業概要を端的に表示。最優先で閲覧するセクション）→ **ミッション・ビジョン**（地域貢献の姿勢と長期ビジョン。報告書や提案資料に引用可能な文言）→ **サービス紹介**（事業ポートフォリオの全体像。「この会社は何ができるのか」の把握）→ **CTA**（メールで連絡。件名テンプレートに「自治体・行政からのお問い合わせ」カテゴリがあると望ましい） |

### 想定フロー

```
業務上の情報収集（DX 推進・補助金周知先リストアップ）
  → 「沖縄 AI 観光」「AI.LandBase」で検索
    → コーポレートサイト（トップ）
      → 会社案内（法人情報確認）→ ミッション・ビジョン → サービス概要
        → メールで連絡 or ブックマークして報告書に記載
```

## ペルソナ間の優先度と対応セクションマトリクス

| セクション | ペルソナ A（宿泊・宿泊税） | ペルソナ B（飲食・効率化） | ペルソナ C（自治体） |
|------------|:-:|:-:|:-:|
| ヒーロー | ◎ 第一印象 | ◎ 共感 | ○ 概要把握 |
| サービス紹介（プラン・価格） | ◎ 費用確認 | ◎ 月額確認 | ○ 事業把握 |
| AI Suite 一覧 | ○ 宿泊税以外の可能性 | ◎ 業務課題との照合 | ○ 技術理解 |
| 会社案内 | ◎ 地元確認 | ○ 信頼確認 | ◎ 法人確認 |
| ミッション・ビジョン | △ 関心低い | ○ 共感 | ◎ 引用材料 |
| CTA（メール） | ◎ 相談 | ◎ 相談 | ◎ 連絡 |

**凡例**: ◎ 最重要 / ○ 重要 / △ 補助的

### Phase 2 への示唆

1. **サービス紹介が最重要セクション**: 3ペルソナとも「この会社は何を提供していて、いくらなのか」を確認しに来る。プラン・価格の明示は最優先
2. **会社案内は信頼の要**: ペルソナ A（地元確認）と C（法人確認）の両方が重視。所在地・代表者・Ikigai Stay の写真は必須コンテンツ
3. **ヒーローのメッセージは汎用性が必要**: 宿泊業特化すぎるとペルソナ B が離脱する。「沖縄の観光業」という上位概念で包括する
4. **AI Suite 一覧は発見の場**: 特にペルソナ B にとって「自分の課題に使えるツールがある」と気づくトリガーになる
5. **CTA は全ペルソナ共通で低障壁**: メール型 CTA は3ペルソナとも受容性が高い。件名テンプレートで問い合わせ種別を分けると、受信側の振り分けにも有効
