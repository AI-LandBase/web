## 1. 設計方針

### トップページ構成方針: ハイブリッド型（LP 風の充実トップ + 個別ページ）

**判断**: 1ページ完結型でも純粋な入口型でもなく、**トップページに主要情報を凝縮しつつ、各セクションから個別ページへの深掘り導線を設ける**ハイブリッド型を採用する。

**判断理由**:

| 観点 | 根拠 |
|------|------|
| ペルソナ A の流入経路 | LP → コーポレートへの遷移。「この会社どんな会社？」を1ページでざっと把握したい。スクロールで完結する LP 的体験が合う |
| ペルソナ B の流入経路 | 指名検索で直接到達。トップで全体像を掴み、興味のあるセクション（サービス・価格）に素早くアクセスしたい |
| ペルソナ C の流入経路 | 指名検索で直接到達。会社案内（法人情報）を最優先で確認するため、個別ページが必要 |
| 競合分析の知見 | カケハシ: ミッション → サービスの接続がトップで完結しつつ、詳細は個別ページ。この構成が AI.LandBase のコンテンツ量に最も適合 |
| コンテンツ量 | サービスは3プラン + AI Suite 10ツール、会社案内は法人情報 + ミッション + 拠点紹介。トップだけでは情報過多になるが、ページ数を増やしすぎると中小経営者が迷う |

**実装への示唆**: トップページのセクションはアンカーリンクで直接遷移可能にし、ナビゲーションからはアンカー（`/#services` 等）と個別ページ（`/services`）の両方にアクセスできる設計とする。

### 全体の設計思想

Phase 1 総括からの接続:

1. **「ミッションを語るが、トップでは抽象に逃げず、現場課題を先に置く」** — カケハシ型のミッション先行ではなく、ペルソナ A・B の具体的な課題を先に示し、その解決手段として AI Suite を提示。ミッション・ビジョンは会社案内に配置
2. **価格の透明性が最大の差別化** — 競合3社が価格非公開の中、プラン・価格を明示する。サービスページで最も目立つ要素にする
3. **地域密着の証明** — 所在地・Ikigai Stay・代表者の情報を、抽象的なミッション文より優先して配置する
4. **CTA のシンプルさと品格の両立** — ELYZA/カケハシの「押しつけがましくない」トーンを踏襲しつつ、AI.LandBase のメール CTA はさらに低障壁

## 2. サイトマップ

### ページ一覧と階層

| パス | ページ名 | 目的 | 階層 | 優先度 |
|------|---------|------|:----:|:------:|
| `/` | トップページ | ヒーロー + 各セクションのサマリー + CTA。「何の会社か」「何ができるか」「いくらか」を1ページで伝える | 1 | 必須 |
| `/services` | サービス | 事業ポートフォリオ全体。3プラン詳細 + AI Suite 10ツール一覧 + 価格 | 1 | 必須 |

### サービス詳細ページの要否

**判断: 作らない**（単一の `/services` ページで完結）

**根拠**:
- プランは3種（スタンダード / プロフェッショナル / AI Suite Server）+ スポット。各プランの説明量はセクション1つ分で収まる
- ペルソナ A・B とも「価格一覧を見比べて自分に合うプランを選ぶ」行動パターン。ページ遷移なしで比較できる方が UX が良い
- 競合3社は複数プロダクトを持つため個別ページが必要だったが、AI.LandBase は「AI Suite」という1パッケージの中のプラン違いであり、構造が異なる
- AI Suite 10ツールの詳細は `/services` ページ内のアコーディオンまたはセクションで展開すれば十分

### 将来の拡張ページ予約

| `/partners` | パートナー | パートナー企業との連携実績が蓄積された段階 | 現時点では連携実績が未公開 |

## 3. グローバルナビゲーション

### ヘッダー構成

```
[ロゴ]  サービス  会社案内  お知らせ*  [お問い合わせ（CTA ボタン）]
```

*「お知らせ」は `/news` ページ追加時に有効化。初期リリースでは非表示。

| 順序 | ラベル | リンク先 | 選定理由 |
|:----:|--------|---------|---------|
| 1 | サービス | `/services` | ペルソナ A・B が最優先で確認するセクション。Phase 1 マトリクスで全ペルソナが重視 |
| 2 | 会社案内 | `/about` | ペルソナ C が最優先、ペルソナ A も「地元確認」で重視。2番目に置くことでバランスを取る |
| 3 | お問い合わせ | `/contact` | 右端に CTA ボタンとして配置。競合3社共通のパターンに準拠 |

**項目数を3つに絞る理由**:
- ペルソナ A・B はデジタルリテラシーが中〜低。選択肢が多いと迷う
- 競合分析で tripla のナビは9項目あるが、上場 SaaS としてのコンテンツ量あってこそ。AI.LandBase の現フェーズでは3項目で十分
- ELYZA のシンプルなナビ構成に近い思想

**ヘッダーの挙動**:
- スクロール時にヘッダー固定（sticky）— 競合3社すべてが採用しているパターン
- CTA ボタンは常に視認可能

### フッター構成

```
[ロゴ + コピーライト]

サービス          会社案内          法務情報              お問い合わせ
- スタンダード    - 会社概要        - プライバシーポリシー  - info@ai-landbase.jp
- プロフェッショナル - ミッション・ビジョン - 特定商取引法に基づく表記  （JS動的生成）
- AI Suite Server - アクセス
- スポット発注

© 2025 株式会社 AI.LandBase
```

**ヘッダーとフッターの役割分担**:

| 要素 | ヘッダー | フッター |
|------|---------|---------|
| 役割 | 主要導線に絞った高速ナビゲーション | 全ページへのアクセスを網羅する補完的ナビゲーション |
| 項目数 | 3項目（サービス / 会社案内 / CTA） | 全ページ + サブセクションへの直リンク |
| 法務ページ | 含まない | 含む（プライバシーポリシー / 特商法） |
| CTA | ボタン型（目立つ） | テキストリンク + メールアドレス表示 |

### ナビラベル選定理由

| 確定ラベル | 検討した候補 | 選定理由 |
|-----------|------------|---------|
| **サービス** | プロダクト / ソリューション / 事業内容 | ペルソナ A・B の語彙に最も近い。「プロダクト」は IT 系、「ソリューション」は大企業向けの響き。中小経営者には「サービス」が自然 |
| **会社案内** | 会社情報 / 私たちについて / About | ペルソナ C（行政）が使い慣れた公式表現。「私たちについて」はカジュアルすぎ、「About」は英語表記がターゲットに合わない |
| **お問い合わせ** | ご相談 / お気軽にご連絡ください / Contact | 「お問い合わせ」が最も汎用的で3ペルソナ全員に通じる。CTA ボタン内のラベルは「お問い合わせ」、ボタン下の補足テキストで「お気軽にメールでご相談ください」と添える |

## 5. CTA 配置戦略

### 主 CTA 配置マップ

| 配置場所 | ページ | 形式 | 件名プリセット |
|---------|--------|------|--------------|

### ペルソナ C 向け CTA の優先配置

ペルソナ C（自治体担当者）は会社案内を最優先で閲覧する。CTA の優先配置:

1. **`/about` ページ末尾** — 法人情報を確認した直後に「自治体・行政からのお問い合わせ」が表示される
2. **`/contact` ページの件名テンプレート** — 「自治体・行政」カテゴリを上位（3番目）に配置し、視認性を確保
3. **トップページ末尾 CTA セクション** — 件名テンプレート選択式にすることで、ペルソナ C も適切なカテゴリを選択可能

### 弱い CTA の方針

BRIEF でフォーム・資料請求は不採用のため、メール以外の弱い CTA の選択肢は限定的。以下の方針で対応する:

| 弱い CTA | 配置場所 | 目的 |
|----------|---------|------|
| AI Suite 10ツール一覧の閲覧誘導 | トップページ サービス概要セクション | 問い合わせ前に「もっと知りたい」を満たす。情報収集段階のペルソナ B に対応 |
| 宿泊税 LP への誘導リンク | `/services` AI Suite Server プラン内 | 「補助金で実質0円」の詳細は LP で確認。高 intent のペルソナ A を LP → 問い合わせへ誘導 |
| 会社案内への誘導 | トップページ 実証紹介セクション | 信頼性を深掘りしたいペルソナ C を会社案内へ誘導 |

**制約下での設計判断**: 資料請求や無料トライアルといった一般的な弱い CTA が使えないため、「情報の深掘り導線」（ページ内遷移・LP 遷移）を弱い CTA の代替とする。これにより、問い合わせの準備ができていない訪問者にも次のアクションを提示し、離脱を防ぐ。

### CTA の温度感設計

```
[高 intent] メールで相談（主 CTA）
     ↑
[中 intent] 宿泊税 LP で詳細確認 / サービスページで価格確認
     ↑
[低 intent] AI Suite ツール一覧を閲覧 / 会社案内で信頼性確認
     ↑
[情報収集] トップページをスクロールして全体像を把握
```
